# Next.js Forum Project - Tech Stack & Plan

## Project Overview
A modern forum application built with a cutting-edge frontend and a scalable backend architecture. 

## Selected Tech Stack

1. **Core Framework**
   - Next.js 16 (App Router)
   - React 19 (Server Components, Server Actions)
   - TypeScript

2. **Database & ORM**
   - Drizzle ORM (Offers great type safety and easy migration capabilities)
   - Database: SQLite (for rapid local development/prototyping)
   - Future Database: PostgreSQL (planned for production scaling; Drizzle makes this transition straightforward)

3. **Authentication**
   - BetterAuth (https://better-auth.com/)
   - Why: Lightweight, highly customizable, and integrates seamlessly with Drizzle.

4. **UI & Styling**
   - Tailwind CSS: For rapid utility-first styling
   - Shadcn-UI: For accessible, unstyled, and customizable UI components
   - Lucide Icons: Clean typography and icons

5. **State Management & Data Fetching**
   - TanStack Query (React Query): For optimistic UI updates (e.g., instant upvotes) and infinite scrolling for feed.
   - Zustand: For lightweight global UI state (e.g., managing modals or sidebars).

6. **Rich Text Editor**
   - Tiptap: A headless, highly customizable rich text editor. Perfect for creating full-featured forum threads and comments without being locked into a specific UI. For image handling in editor, implement an upload endpoint that saves images to `/public/uploads` (MVP); return the public URL to tiptap.

7. **Media Chooser**: 
   - centralized upload and choose images (link) for tiptap editor (please use dialog shadCN UI). so when uploading images its just store link to postgres/sqlite and image to folder uploads, but when i want to add image to tiptap editor, i just get that link from media chooser. 8. **File Storage**: 
   - local `public/uploads` for MVP


## Architecture Notes
- Use Next.js Server Components for reading data directly from the database to ensure fast initial page loads and excellent SEO.
- Use Next.js Server Actions for form submissions and mutations.
- React Query will handle client-side interactivity where optimistic updates or infinite pagination is required.
